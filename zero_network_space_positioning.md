# 2. 产品定位（One-liner）

**零网络空间是一款完全离线、无网络权限的本地加密隐私资料库，用于存放你的私密照片、视频、文件与笔记。**

---

# 3. 产品使命（Mission）

> **让每个人都拥有一个不依赖网络、不受监控、不泄露隐私的真正私人空间。**

在一个所有 App 都在“云端化、账号化、同步化”的时代，我们选择相反的方向：  
**用户的数据不应该被上传、不应该被分析、不应该被追踪。**

零网络空间的使命就是：  
**打造一个“无法联网、无法上传、无法泄露”的终极离线隐私工具。**

# 4. 面向人群（Target Users）

## 核心用户群
1. **重隐私用户**  
   - 不相信 iCloud/Google Photos  
   - 担心照片、视频被上传或误同步

2. **日常私密内容存储者**  
   - 情侣照片  
   - 个人证件  
   - 账单、合同等敏感文件

3. **工作敏感行业用户**  
   - 律师 / 医疗 / 银行 / 政府 / 新闻行业

4. **极客与数据主权支持者**  
   - 强烈强调数据本地化与掌控权

5. **需要分级访问控制的用户**（新增）
   - 需要临时展示部分内容，但不暴露敏感资料
   - 希望在不同场景下显示不同内容级别
   - 需要向他人展示App但保护核心隐私

## 用户需求痛点
- 担心“零网络空间”实际上在悄悄上传数据
- 不想注册、不想绑定账号、不想被追踪
- 希望照片文件存本地，不走云端
- 需要一个真正安全、可信的隐私空间

---

# 5. 产品核心价值（Value Proposition）

## **1. 完全离线（Zero Network）**
- App 内部无网络代码路径  
- 不请求任何网络权限  
- 不含 SDK、广告、统计、云备份

## **2. 本地加密存储（Zero Upload）**
- 所有内容 AES-256 本地加密  
- 密钥仅存于设备 Keychain

## **3. 零账号、零追踪（Zero Identity）**
- 不注册  
- 不登录  
- 不收集隐私数据

## **4. 安全可靠（Zero Risk）**
- 没有云端 = 没有泄露风险  
- 没有交互上传 = 没有第三方获取可能

## **5. 一次性付费解锁（Zero Subscription）**
- 不做订阅  
- 不做套路收费  
- 付费后永久使用

# 6. 产品核心原则（Product Principles）

以下原则贯穿产品生命周期，不得妥协：

1. **不得接触任何外部服务器**（包括验证、统计、崩溃分析）
2. **不加入任何需要网络的 SDK**（广告、埋点、推送除外——推送不会使用）
3. **不创建用户账号体系**
4. **数据必须全部本地加密**
5. **信息架构极简**，用户能 1 秒理解界面
6. **安全优先于功能**
7. **隐私优先于体验**（例如不允许未授权截图）

# 7.5. V1.2 核心功能亮点：访客模式（Guest Mode）

### **为什么需要访客模式？**

在日常使用中，用户可能遇到以下场景：
- 需要向他人展示App的某些内容，但不想暴露隐藏空间
- 临时借用手机给他人，但不想让对方看到所有私密内容
- 在公共场合使用App，希望有更低风险的访问模式

### **访客模式解决方案**

零网络空间V1.2引入了**双密码体系**：
- **主密码**（6-8位）：完全访问，包括隐藏空间
- **访客密码**（6-8位纯数字）：受限访问，自动隐藏敏感内容

### **核心特性**

1. **双重认证体系**
   - 两个独立密码，互不干扰
   - 访客密码必须为纯数字（易记、易分享）
   - 服务端验证确保两个密码不能相同

2. **智能内容过滤**
   - 访客模式下自动隐藏"隐藏空间"中的所有笔记
   - 保持界面一致性，不暴露隐藏功能的存在
   - 访客无法察觉某些内容被隐藏

3. **安全状态管理**
   - 应用重启后强制重新认证（防止状态泄露）
   - 线程安全的状态管理（@MainActor保护）
   - 完整的生命周期管理（防止崩溃）

4. **用户体验优化**
   - 简单的设置流程（设置界面一键配置）
   - 清晰的模式指示（用户知道当前处于哪种模式）
   - 完整的国际化支持（英文/中文）

### **使用场景举例**

**场景1：向朋友展示App**
```
你想向朋友介绍零网络空间，但你的隐藏空间里有私密日记。
→ 给朋友提供访客密码
→ 朋友可以看到你的照片和文件，但看不到隐藏空间
→ 你的隐私得到完整保护
```

**场景2：公共场合使用**
```
在咖啡厅或办公室使用App时，担心有人看到敏感内容。
→ 使用访客密码登录
→ 正常使用大部分功能，但核心隐私内容自动隐藏
→ 降低泄露风险
```

**场景3：临时借用**
```
家人需要借用你的手机查看某个文件。
→ 提供访客密码
→ 家人可以完成任务，但无法访问你的私密空间
→ 保持家庭和谐的同时保护隐私
```

### **技术实现亮点**

- **企业级安全标准**：通过AI代码审查评分，修复所有严重和高优先级安全问题
- **架构优化**：采用现代SwiftUI最佳实践（环境对象依赖注入、主线程保护）
- **零网络承诺**：访客模式完全本地实现，无任何网络请求

# 8. 差异化（Differentiators）

| 特性 | 普通隐私App | 零网络空间 |
|------|---------------|--------------|
| 是否联网 | 是（广告/统计/云） | **否（代码级阻断）** |
| 隐私数据 | 可能会上传 | **100% 本地** |
| SDK | 有广告/埋点 | **无任何网络 SDK** |
| 注册 | 需要账号 | **无需注册** |
| 数据控制权 | 服务端存部分数据 | **完全在用户手里** |
| 访问控制 | 单一密码 | **双密码体系（主/访客）** |
| 内容隔离 | 无或简单隐藏 | **智能分级访问控制** |
| 收费模式 | 广告/订阅 | **一次性买断** |
| 信任来源 | 品牌承诺 | **技术实现 + 零网络** |

> **用户完全掌控的离线数字隐私生态。**
